# Contributing to Phant

Thank you for your interest in contributing to Phant! Your help is greatly appreciated. This guide will help you get started and ensure a smooth contribution process.

## How to Contribute

1. **Fork the repository**
   - Click the "Fork" button at the top right of the [Phant GitHub page](https://github.com/phant-app/phant).
2. **Clone your fork**
   - `git clone https://github.com/your-username/phant.git`
3. **Create a feature branch**
   - `git checkout -b my-feature`
4. **Make your changes**
   - Follow the code style and best practices used in the project.
   - Add or update tests as needed.
5. **Commit your changes**
   - Write clear, descriptive commit messages.
6. **Push to your fork**
   - `git push origin my-feature`
7. **Open a Pull Request**
   - Go to your fork on GitHub and click "Compare & pull request".
   - Fill in a clear title and description for your PR.

## Code Style & Guidelines

- Follow the existing code style for Go, TypeScript, and React code.
- Use `gofmt` for Go files and `npm run format` for frontend code.
- Write clear, maintainable, and well-documented code.
- Add or update tests for new features and bug fixes.
- Keep pull requests focused and minimal—one feature or fix per PR is best.

## Reporting Issues

If you find a bug or have a feature request, please [open an issue](https://github.com/phant-app/phant/issues) and provide as much detail as possible:
- Steps to reproduce
- Expected and actual behavior
- OS and Phant version
- Logs or screenshots if relevant

## Code of Conduct

Please be respectful and considerate in all interactions. We follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/).

## Getting Help

- Check the [documentation](docs/) and [GitHub Discussions](https://github.com/phant-app/phant/discussions).
- For setup or development questions, open a discussion or issue.
